## v0.1.2 — 버그 수정 (2026-03-27)

### 수정된 버그

#### 1. 카테고리 시설 마커 미표시 (Critical)
- **원인**: API가 `lat`/`lng` 필드를 반환하는데, SeoulMap에서 `p.latitude`/`p.longitude`로 접근 → 전부 `undefined`로 필터링됨
- **수정**: `SeoulMap.jsx` — `p.lat || p.latitude` 양쪽 필드명 모두 지원
- **추가 수정**: 다중 카테고리 선택 시 각 카테고리별 병렬 fetch 후 병합

#### 2. 지하철 노선 선택 팝업 미표시 (UI)
- **원인**: 사이드바 `aside`에 `overflow-y-auto` → CSS 스펙상 `overflow-x`도 `auto`로 강제 → absolute 드롭다운 잘림
- **수정**: `SubwayFilter.jsx` — `position: fixed` + `getBoundingClientRect()` 기반 좌표, z-index 9999

#### 3. Vite 프록시 포트 충돌 (Infra)
- **원인**: `vite.config.js` 프록시가 `8001`(crypto 백엔드)을 바라봄
- **수정**: 프록시 타겟 `8001` → `8000`, 3개 프로젝트 포트 고정 (`strictPort: true`)

### 포트 배정표

| 프로젝트 | 프론트엔드 | 백엔드 |
|----------|-----------|--------|
| seoul-culture-map | `:5173` | `:8000` |
| crypto-volatility-dashboard | `:5174` | `:8001` |
| portfolio | `:5175` | — |

## v0.3.0 — Analytics 지도 기반 분석 (2026-03-27)

**기존 차트 위젯 → 지도 위 직접 분석 시각화로 전환**

### 4가지 분석 모드 (지도 마커로 표현)

1. **군집분석** — K-means 클러스터별 색상 마커 + 사이드바 범례
2. **권역별** — 5대 권역 색상 마커 + 하단 권역별 시설 요약 카드 (Top 3 카테고리)
3. **카테고리 밀도** — 카테고리 선택 → 시설 수에 따라 마커 색상(연노랑→빨강)/크기 변화
4. **지하철 접근성** — 반경 1.5km 내 역 수 계산 → 빨강(적음)~초록(많음) 마커 + Top 10 랭킹

### 기존 차트 유지 (사이드바)
- 자치구 비교 레이더 차트
- 전체 카테고리 분포 파이 차트
- 자치구 클릭 시 상세 카테고리 바 차트 팝업

### 기술적 의사결정
- 하드코딩 카테고리 라벨 제거 → API 데이터에서 자동 추출 (`useMemo`)
- Haversine 거리 계산을 프론트에서 수행 (지하철역 수백 개 수준이라 성능 문제 없음)

## v0.4.0 — 관광 코스 추천 (2026-03-27)

**Course 페이지를 자치구 단위 → 목적별 추천으로 재구성**

### 5가지 관광 목적
| 목적 | 색상 | 관련 카테고리 |
|------|------|--------------|
| 공연 중심 | 핑크 | 공연시설, 문화시설 |
| 자연 힐링 | 초록 | 공원, 레포츠 |
| 역사 탐방 | 보라 | 박물관/유적지, 관광지 |
| 액티비티 | 노랑 | 레포츠, 공원 |
| 문화 예술 | 인디고 | 문화시설, 공연시설, 관광지 |

### 동작 흐름
1. 목적 선택 → 자치구별 관련 카테고리 시설 합산 점수 계산
2. Top 5 자치구가 지도에 순위 마커로 표시 (1등이 가장 큼)
3. 자치구 클릭 → 해당 카테고리 시설만 지도에 표시
4. AI 코스 추천 버튼 → OpenAI 기반 반나절 코스 생성 (fallback 지원)

## v0.5.0 — 즐겨찾기 시스템 리뉴얼 + 시설 이미지 (2026-03-27)

### 즐겨찾기 시스템 v2
- **기존**: 문자열 ID (`"강남구-공연시설"`) → 위치 정보 없음
- **변경**: 시설 객체 저장 (`{ id, name, category, district, lat, lng }`)
- 스토리지 키 `v2`로 변경 (기존 데이터 충돌 방지)

### Culture Map 시설 팝업
- 시설 마커 클릭 → **사진 + 시설명 + 카테고리 + 즐겨찾기 버튼** 팝업
- 이미지가 없는 시설은 텍스트 팝업만 표시
- DistrictPopup에서 카테고리별 하트 버튼 제거 (개별 시설 단위로 변경)

### Favorites 페이지
- 저장된 시설이 **지도 위 하트 마커**로 표시 (카테고리별 색상)
- 오른쪽 패널에서 시설 클릭 → 지도 해당 위치로 flyTo
- X 버튼으로 개별 해제

### 한국관광공사 시설 이미지 연동
- Place 모델에 `image_url` 컬럼 추가
- Tour API sync 시 `firstimage`/`firstimage2` 필드 저장
- **2,529개 시설 중 1,177개(47%)에 이미지 보유**
- 이미지 소스: `tong.visitkorea.or.kr` (관광지, 문화시설, 공연시설, 레포츠)
- 서울 공공데이터(문화공간/공원)는 이미지 미제공 → `null`

### DB 마이그레이션
- SQLite에 `ALTER TABLE places ADD COLUMN image_url TEXT` 직접 실행
- `POST /api/sync` 호출로 Tour API 데이터 재수집 (이미지 포함)
